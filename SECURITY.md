# Security policy

Please use GitHub's private reporting flow for anything that may affect the confidentiality, integrity, or availability of Pastel or a server managed by it. Open a [private vulnerability report](https://github.com/iamkaf/pastel/security/advisories/new) with the affected version, impact, reproduction steps, and any suggested mitigation. Do not open a public issue for a suspected vulnerability.

Use [GitHub Issues](https://github.com/iamkaf/pastel/issues) for reproducible non-security bugs and [GitHub Discussions](https://github.com/iamkaf/pastel/discussions) for support, setup questions, and ideas.

Security fixes target the latest release. If a report affects older releases too, include that information so the advisory can describe the complete affected range.
